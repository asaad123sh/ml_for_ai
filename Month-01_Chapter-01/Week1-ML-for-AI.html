Subtopic 2
data types
Concept Focus
  • WHAT: Categories that tell Python what kind of data a variable holds.
  • WHAT (continued): Main types: - int → whole numbers: 5, 100, -3 - float → decimal numbers: 3.14, 0.001 - str → text: "hello", "ML is great" - bool → True or False only - NoneType → represents nothing/empty
  • WHY: Python handles numbers, text, and logic differently.
  • WHY (continued): Wrong data type = wrong result or error
  • WHERE: Every calculation, every model input, every comparison
  • KEY: "The type decides what operations are allowed"
  • HOW TO APPLY: Create values -> check with type() -> convert with int()/float()/str() when required.
  • USE IN PRACTICE: You use types to avoid bugs and to convert input safely.
  • COMMON MISTAKE: Most beginner bugs come from mixing string and number types.
  • EXAMPLE: Simple example: convert '25' to int and '3.5' to float, then print type().
  • REMEMBER THIS: The type decides what operations are allowed
Example 1: Core Example (Python)
# New built-in: type() -> type() tells you the data type of a value.

# Data types tell Python what kind of value this is
count = 5                  # int
price = 99.5               # float
course = 'ML Basics'       # str
is_active = True           # bool

# type() checks the data type
print(type(count), type(price), type(course), type(is_active))
