Gradient boosting
Separate Subtopic Learning Modules
Subtopic 1
Gradient boosting
Concept Focus
  • WHAT: Boosting method where each new tree is trained to predict the RESIDUALS (errors) of the current ensemble.
  • WHAT (continued): Uses gradient descent in function space.
  • WHY: Frames boosting as gradient descent optimization.
  • WHY (continued): Can work with any differentiable loss function.
  • WHERE: Tabular data, almost always top competitor
  • KEY: "Build trees to predict what the current model got wrong"
  • HOW TO APPLY: Start with tiny numbers -> compute step-by-step -> verify with code.
  • USE IN PRACTICE: You use it to understand training logic, metrics, and optimization.
  • COMMON MISTAKE: Understand meaning first, formula second. Do not memorize without intuition.
  • EXAMPLE: Simple example: use small integers so every step can be checked manually.
  • REMEMBER THIS: Build trees to predict what the current model got wrong
Example 1: Core Example (Python)
# Gradient descent intuition on f(w) = w^2
w = 5.0
learning_rate = 0.1

for step in range(5):
    grad = 2 * w               # Derivative of w^2
    w = w - learning_rate * grad
    print(f'Step {step + 1}, w = {w:.4f}')

Step 5, w = 1.6384
