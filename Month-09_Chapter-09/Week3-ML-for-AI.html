Vanishing gradient problem
Separate Subtopic Learning Modules
Subtopic 1
Vanishing gradient problem
Concept Focus
  • WHAT: During backpropagation through many time steps, gradients get multiplied repeatedly.
  • WHAT (continued): If gradient < 1 at each step, it shrinks exponentially → becomes ≈ 0 at early steps.
  • WHY: Plain RNNs can't handle long sequences (50+ words) because gradient signal disappears.
  • WHY (continued): "What was said 100 words ago has zero gradient signal in vanilla RNN"
  • WHERE: Why LSTM and GRU were invented
  • KEY: "Gradients die over many time steps — model forgets long ago"
  • HOW TO APPLY: Start with tiny numbers -> compute step-by-step -> verify with code.
  • USE IN PRACTICE: You use it to understand training logic, metrics, and optimization.
  • COMMON MISTAKE: Understand meaning first, formula second. Do not memorize without intuition.
  • EXAMPLE: Simple example: use small integers so every step can be checked manually.
  • REMEMBER THIS: Gradients die over many time steps — model forgets long ago
Example 1: Core Example (Python)
# Gradient descent intuition on f(w) = w^2
w = 5.0
learning_rate = 0.1

for step in range(5):
    grad = 2 * w               # Derivative of w^2
    w = w - learning_rate * grad
    print(f'Step {step + 1}, w = {w:.4f}')

LSTM
Concept Focus

  • WHERE: Language modeling, machine translation, speech recognition
  • KEY: "RNN with gates that control memory — can remember long-term"
  • HOW TO APPLY: Define network -> pass input -> compute loss -> update weights.
